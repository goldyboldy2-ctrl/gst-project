E-Invoicing Complete Guide 2026

Everything you need to know about mandatory e-invoicing under GST

What is E-Invoicing?

E-invoicing (electronic invoicing) is a system where B2B invoices are authenticated electronically by the GST Network (GSTN) for further use on the GST portal. It's not just uploading invoices - it's a standardized process of generating and reporting invoices in real-time.

Key Points

  • Invoice data is uploaded to Invoice Registration Portal (IRP)
  • IRP validates and returns a unique Invoice Reference Number (IRN)
  • Digitally signed e-invoice + QR code generated
  • Data automatically flows to GST portal and e-way bill system
  • No need to upload invoice details separately in GSTR-1

Important

E-invoicing is NOT the same as e-filing of returns. It's a separate process that must be completed at the time of invoice generation.

How E-Invoicing Works

Step 1: Generate Invoice in Your System

Create invoice in your ERP/accounting software as usual with all mandatory fields.

Step 2: Convert to JSON Format

Invoice data converted to standard JSON schema format as per GSTN specifications.

Step 3: Upload to IRP

JSON file uploaded to Invoice Registration Portal (IRP) via API or portal.

Step 4: IRP Validates & Generates IRN

IRP validates invoice, generates unique 64-character IRN (Invoice Reference Number).

Step 5: Receive Digitally Signed Invoice

IRP returns digitally signed JSON with IRN and QR code.

Step 6: Generate PDF with QR Code

Your system generates final PDF invoice with IRN and QR code embedded.

Step 7: Auto Population

Invoice data automatically flows to:

  • GST portal (auto-populated in GSTR-1)
  • E-way bill system (if applicable)
  • Buyer's GSTR-2B

IRP Registration Process

Step 1: Visit IRP Portal

Go to einvoice.gst.gov.in

Step 2: Login with GST Credentials

Use your GSTIN username and password (same as GST portal)

Step 3: Register on IRP

Click "Register" and accept terms and conditions

Step 4: Add Users

Create sub-users if needed (max 3 users per GSTIN)

Step 5: Generate API Credentials

For API integration: Generate username and password for system-to-system integration

Mandatory Fields in E-Invoice

Supplier Details

  • GSTIN
  • Legal name
  • Trade name (if any)
  • Complete address with PIN code

Buyer Details

  • GSTIN / UIN (for exports)
  • Legal name
  • Complete address with PIN code
  • Place of supply

Invoice Details

  • Invoice number (unique, max 16 characters)
  • Invoice date
  • Invoice type (Regular, Debit Note, Credit Note)
  • Document type (Invoice, Bill of Supply, etc.)

Item Details

  • Item description
  • HSN/SAC code (6 or 8 digits as applicable)
  • Quantity
  • Unit of measurement
  • Unit price
  • Total amount
  • Discount (if any)
  • GST rate
  • GST amount (CGST, SGST/UTGST, IGST, Cess)

Payment Details

  • Total taxable value
  • Total GST amount
  • Total invoice value

Software/Tools for E-Invoicing

Option 1: Direct Portal Upload

Best for: Small businesses with low invoice volume (< 100 invoices/month)

Process: Manually enter invoice details on IRP portal

Cost: Free

Limitation: Time-consuming, prone to errors

Option 2: Excel/Bulk Upload

Best for: Businesses with moderate volume (100-500 invoices/month)

Process: Fill Excel template, upload to IRP

Cost: Free

Limitation: Manual data entry, limited automation

Option 3: GST Suvidha Providers (GSPs)

Best for: Medium to large businesses

Process: Third-party software that integrates with your ERP

Cost: Subscription-based (‚Çπ500-5,000/month depending on volume)

Examples: ClearTax, Tally, Busy, SAP, etc.

Benefits: Full automation, error handling, bulk processing

Option 4: API Integration

Best for: Large enterprises with high volume

Process: Direct API integration between your ERP and IRP

Cost: Development cost + maintenance

Benefits: Fully automated, real-time, seamless

Understanding IRN (Invoice Reference Number)

IRN is a unique 64-character hash generated by IRP for each invoice.

Sample IRN

db78c3d7e18f8b5e9c3a6b2f4d1e8a7c9b5f3e7d1a4c8b2e6f9d3c7a1b5e4f8d2

Once generated, IRN cannot be changed or modified

Important Points About IRN

  • Valid for entire financial year
  • Cannot be regenerated for same invoice details
  • Must be printed on invoice copy
  • Used for tracking and verification
  • Part of QR code on invoice

QR Code on E-Invoice

Every e-invoice must have a QR code containing:

  • Supplier GSTIN
  • Buyer GSTIN (if registered)
  • Invoice number
  • Invoice date
  • Invoice value
  • Number of line items
  • HSN code of main item
  • IRN
üì±

QR Code Benefits

Buyers and tax officers can scan QR code to instantly verify invoice authenticity on IRP portal. Reduces fake invoice risk.

E-Invoice Cancellation

If invoice is generated by mistake or contains errors:

Cancellation Window

Time Limit: Within 24 hours of IRN generation

Process: Login to IRP portal ‚Üí Cancel E-Invoice ‚Üí Enter IRN ‚Üí Select reason ‚Üí Submit

Reasons: Duplicate, data entry error, order cancelled, etc.

After 24 Hours

Cannot cancel IRN. Must issue credit note to reverse the transaction. Credit note must also be e-invoiced.

E-Invoice and E-Way Bill Integration

E-invoicing is integrated with e-way bill system:

How It Works

  • When e-invoice is generated, Part A of e-way bill is auto-populated
  • Only need to fill Part B (transporter details) if goods movement required
  • Saves time and eliminates double data entry
  • Reduces errors in e-way bill generation

Common Challenges & Solutions

Challenge 1: Technical Errors

Issue: API timeout, network errors, portal downtime

Solution: Use multiple IRP portals (5 portals available), retry mechanism, maintain offline backup

Challenge 2: Software Integration

Issue: Existing ERP doesn't support e-invoicing

Solution: Get software update, use GSP middleware, or change to compliant software

Challenge 3: Bulk Invoice Processing

Issue: Generating IRN for thousands of invoices daily

Solution: API integration, bulk upload via JSON, automated workflows

Challenge 4: Training Staff

Issue: Employees not familiar with new process

Solution: Conduct training sessions, create SOPs, test environment practice
